Swagger ⏰約20分

🌞 Swaggerとは？

  • アプリで使うAPI（データのやりとりのルール）を設計・確認するためのツールです。
  • 「どんなURLに」「どんなデータを送ると」「何が返ってくるか」を、コードを書く前に決めることができます。
  • Figmaが「画面の設計」だとすると、Swaggerは「データの設計」を行う場所です。
  • バックエンドとフロントエンドの共通ルールを決める役割があります。

🌞 Swaggerを使う準備をしよう

🖊 Swagger Editorを開こう

下のリンクをクリックしてください
→📚 Swagger Editor

  • 左：Swagger editor → 設定を書く場所。コードで確認。
  • 右：Swagger UI → 設定内容が反映されたAPIドキュメント。見た目で確認。

🌞 APIの基本構造を理解しよう

🖊 パスとは何かを知ろう

  • パスはサーバー上のデータや機能の住所のようなものです。

例：

  • パス：/memo
  • 意味：「メモのデータがある場所」

アプリやブラウザは、この住所を指定して「この場所の情報をください」とサーバーにリクエストします。

🖊 操作の種類（HTTPメソッド）を知ろう

  • パスは「操作の種類」と組み合わせて使います。
メソッド 意味
GET データを取得する（見るだけ）
POST 新しいデータを作る
PUT データを更新する（書き換える）
DELETE データを削除する

※ 今回使用するのは GET と PUT です。

🌞 Swagger EditorでAPIを書いてみよう

🖊 ediorを空にします

  • これから、コードを少しずつコピーしながらUIの変化を確認していきます。
  • 途中でエラーが出ても、最終的に解消されるのでそのまま進めてください。
    （最後までエラーが消えない場合は、ページを再読み込みしてください。）

🌞 APIの情報を定義しよう

🖊 ① openapi / info を書こう

  • APIの名前や説明を書く場所です。
  • Swagger UIの一番上に表示されます。

コピペ用↓

openapi: 3.0.0
info:
  title: メモAPI
  version: 1.0.0
  description: |
    簡単なメモアプリ用のAPIです。
    1つのメモデータを取得・更新することができます。

🌞 APIをグループ分けしよう

🖊 ② tags を書こう

  • APIをカテゴリごとにまとめるための設定です。
  • Swagger UIに「memo」という見出しが表示されます。

コピペ用↓

tags:
  - name: memo
    description: メモ内容の取得・更新

🌞 メモAPIの処理を書こう

🖊 ③ メモを取得するAPI（GET /memo）

  • 保存されているメモを取得するAPIです。
  • Swagger UIでは青色のGETとして表示されます。

コピペ用↓

paths:
  /memo:
    get:
      tags:
        - memo
      summary: 現在のメモ内容を取得する
      description: 保存されている最新のメモ内容を取得します。
      responses:
        "200":
          description: メモの取得に成功しました
          content:
            application/json:
              schema:
                $ref: "#/components/schemas/Memo"

🖊 ④ メモを更新するAPI（PUT /memo）

  • メモの内容を書き換えるAPIです。
  • Swagger UIでは黄色のPUTとして表示されます。

コピペ用↓ ⚠インデントに注意！

    put:
      tags:
        - memo
      summary: メモ内容を更新する
      description: 指定されたメモ内容で /memo リソースを上書きします。
      requestBody:
        required: true
        content:
          application/x-www-form-urlencoded:
            schema:
              type: object
              required:
                - text
              properties:
                text:
                  type: string
                  description: 保存するメモのテキスト（空文字不可）
                  example: "買い物リスト：りんご、牛乳"
      responses:
        "200":
          description: メモの更新に成功しました
          content:
            application/json:
              schema:
                $ref: "#/components/schemas/Memo"
        "400":
          description: textが空
          content:
            text/plain:
              schema:
                type: string
                example: "メモを入力してください"

🌞 データの形を定義しよう

🖊 ⑤ components（Memoの形）を書こう

  • APIで扱うデータの構造を定義します。
  • Swagger UIの下部に「Memo」が表示されます。

コピペ用↓ ⚠インデントに注意！

components:
  schemas:
    Memo:
      type: object
      properties:
        text:
          type: string
          example: "買い物リスト：りんご、牛乳"
      required:
        - text

🌷完成！
